理解 AOP

发表于 2017-02-21   |   分类于 Java   |     |   浏览 次

最近在看《Spring 揭秘》这本书，看到 Spring AOP 框架这一章节，借此机会总结一下 AOP 的相关内容。

AOP 概述

AOP 的全称是 Aspect-Oriented Programming， 中文通常称为面向切面编程。系统开发的目的是为了解决各种需求，其中包括业务需求和系统自身需求。通常我们使用面向对象的方法，对业务需求等普通关注点进行很好的抽象和封装，并使之模块化。但是，生产环境为了对系统进行监控，我们需要对业务需求的实现添加日志记录功能或者业务方法的执行需要一定权限。当系统中的业务对象很多时，我们是否需要在每个业务需求的实现类中加入类似的逻辑，显然，这样做加大了开发和维护的难度。
此时，AOP 的开发方式便可以解决业务需求和系统需求不对称的情况，AOP 引入了Aspect 的概念，用来以模块化的方式对日志记录、安全检查等横切关注点进行封装。我们便可以将 Aspect 横切到个业务模块之上，AOP 便是对 OOP 的一种补充。
